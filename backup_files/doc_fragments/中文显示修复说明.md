# 监控画面中文显示乱码修复说明

## 问题描述

原监控系统的实时分析画面显示中文时出现乱码，这是因为 OpenCV 的 `cv2.putText` 函数不支持中文字符，导致中文显示为方块或乱码。

## 修复方案

使用 PIL (Python Imaging Library) 来处理中文文字渲染，然后将结果转换为 OpenCV 格式。

## 修复的文件

### 1. realtime_camera_display.py
- **功能**：实时摄像头显示 + VLM分析系统
- **修复内容**：
  - 添加了PIL导入和中文字体支持
  - 新增 `load_chinese_font()` 方法加载系统中文字体
  - 新增 `draw_text_chinese()` 方法绘制中文文字
  - 修改 `draw_info_overlay()` 函数使用中文字体绘制

### 2. robot_vision_main.py
- **功能**：机器人视觉识别系统主程序
- **修复内容**：
  - 添加了PIL导入和字体支持
  - 新增中文字体加载和绘制功能
  - 修改 `_prepare_display_frame()` 函数中的文字显示部分

### 3. opencv_gui_monitor.py
- **功能**：基于OpenCV GUI的监控系统
- **修复内容**：
  - 添加了PIL导入和字体支持
  - 新增中文字体处理功能
  - 重写 `create_info_overlay()` 函数使用中文字体

## 依赖库

确保已安装以下Python库：
- PIL (Pillow) - 用于中文字体处理
- OpenCV - 图像处理
- NumPy - 数值计算

安装命令：
```bash
pip install Pillow opencv-python numpy
```

## 注意事项

1. **字体路径**：程序会自动检测系统中可用的中文字体
2. **性能优化**：字体加载仅在初始化时进行一次
3. **降级兼容**：如果中文字体不可用，会自动回退到默认字体
4. **描边效果**：添加了黑色描边增强文字可读性

## 故障排除

如果中文仍显示异常：
1. 检查系统是否安装了中文字体
2. 运行 `test_chinese_display.py` 进行诊断
3. 查看控制台输出的字体加载信息
4. 确认PIL库正确安装：`python3 -c "from PIL import Image; print('PIL OK')"`
